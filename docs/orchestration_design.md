# Multi-Agent Orchestration Design

The `vs-brain` framework employs a hierarchical orchestration pattern to handle complex, multi-step tasks. This system features an **Orchestrator** agent and an **Assistant** agent working together to coordinate specialized agents.

## Core Roles

### 1. The Orchestrator
The Orchestrator is the high-level strategist responsible for:
*   **Goal Analysis**: Understanding the overall objective and its requirements.
*   **Task Decomposition**: Breaking down the main goal into a sequence of sub-tasks.
*   **Agent Selection**: Identifying the most suitable specialized agent for each sub-task.
*   **Workflow Monitoring**: Tracking progress and handling any deviations or failures.
*   **Final Synthesis**: Combining the outputs of specialized agents into a final result.

### 2. The Assistant
The Assistant acts as the Orchestrator's primary deputy, handling:
*   **Inter-Agent Communication**: Facilitating the exchange of information between the Orchestrator and specialized agents.
*   **State Management**: Maintaining a shared context or "ledger" of the workflow's progress.
*   **Sub-task Refinement**: Helping to clarify or refine sub-tasks before they are delegated.
*   **Preliminary Review**: Performing initial checks on the outputs of specialized agents.

## Interaction Pattern

The orchestration follows a structured interaction pattern:
1.  **User Input**: The Orchestrator receives a complex task from the user.
2.  **Decomposition**: The Orchestrator, with help from the Assistant, decomposes the task into a plan.
3.  **Delegation**: The Assistant manages the handoff of sub-tasks to specialized agents (e.g., Information Gathering Agent).
4.  **Execution**: Specialized agents perform their tasks within their own agentic loops, governed by the harness.
5.  **Observation & Feedback**: Results are fed back to the Assistant, which updates the shared state and notifies the Orchestrator.
6.  **Iteration**: The Orchestrator adjusts the plan if necessary and proceeds to the next step.
7.  **Completion**: Once all sub-tasks are finished, the Orchestrator synthesizes the final output.

## Shared State and Memory

The Orchestrator and Assistant utilize a shared memory space (e.g., a specific file or data structure) to track:
*   The current overall plan.
*   The status of each sub-task (e.g., Pending, In Progress, Completed, Failed).
*   The outputs and artifacts generated by specialized agents.
*   The history of agent interactions.

## Security and Governance

The orchestration system operates within the `vs-brain` harness, ensuring that all agent actions—including delegation and communication—are subject to permission checks and budget constraints.
